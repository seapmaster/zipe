Краткое описание и инструкция по применению

Справочник, содержащий описания предметов снабжения.

Текстовая информация по ПС заполняется при переходе по ссылке «Основное», графическая информация заполняется при переходе по ссылке «Присоединенные файлы». Основная информация по ПС разбита на несколько групп, представленных на следующих вкладках ссылки «Основное»:

- «Общие сведения»;

- «Характеристики»;

- «Правила поставки»;

- «Поставщики»;

- «Аналоги»;

- «Цены внешние»;

- «Цены внутренние»;

- «Состав»;

- «Ресурсы и сроки службы»;

- «Регламент ТО»;

- «Служебные данные идентификации и кодирования ПС»;

- «Данные от КСУ НСИ». 

На вкладке «Общие сведения» для нового элемента ПС заполняются следующие поля :

- «Номер ОСК» – символьное значение длиной 14 символов формируется автоматически при установлении статуса записи «Проверено ОСК», уникальное в рамках каталога ПС, обязательное поле при указанном статусе. Маска значения – ОСК‑ХХХХХХХХХХ, где Х –  цифры с недостающими ведущими нулями;

- «Наименование» – наименование ПС по документации разработчика на русском языке. Символьное значение длиной не более 150 символов вводится с клавиатуры, уникальное в рамках каталога кораблей совместно с обозначением, обязательное поле;

- «Обозначение» – обозначение ПС по документации на русском языке. Символьное значение длиной не более 100 символов вводится с клавиатуры;

- «Обозначение (транс.)» – обозначение ПС по документации разработчика транслитерированное. Символьное значение длиной не более 100 символов формируется автоматически по обозначению, с возможностью корректировки;

- «Шифр/марка» – тип, шифр, марка, индекс, типоразмер, условное наименование ПС. Символьное значение длиной не более 150 символов вводится с клавиатуры;

- «УН» – утвержденное наименование ПС. Ссылка на элемент справочника, значение выбирается из связанного справочника «Утвержденные наименования ПС» вручную или заполняется из файла АО «РОЭ»;

- «Код УН» – ссылка на элемент справочника «Утвержденные наименования ПС» заполняется автоматически после заполнения поля «УН»;

- «Самостоятельная поставка» – булевое значение вводится вручную;

- «Документ на поставку» – обозначение документа, по которому поставляется или изготавливается ПС. Символьное значение длиной не более 50 символов вводится с клавиатуры;

- «Тип документа» – тип документа, по которому поставляется или изготавливается ПС. Ссылка на элемент справочника «Тип документации для изготовления и поставки ПС»;

- «Дополнительная информация» – символьное значение неограниченной длины вводится с клавиатуры;

- «Единица измерения» – ссылка на элемент справочника, значение выбирается из связанного справочника «ОКЕИ», обязательное поле;

- «ФНН» – символьное значение длиной 15 символов вводится с клавиатуры или загружается из файла МО РФ. Маска ввода значения – ХХХХ-ХХ-ХХХХХХХ, где Х – цифры;

- «NSN» – символьное значение длиной 13 символов вводится с клавиатуры или загружается из файла АО «РОЭ»;

- «Код INCAT» – символьное значение длиной не более 32 символов формируется автоматически по коду NSN, ФНН, транслитерированному обозначению или вводится вручную;

- «Класс ЕКПС» – ссылка на элемент справочника, значение выбирается из связанного справочника «ЕКПС» или загружается из файла АО «РОЭ», МО РФ;

- «Группа ЕКПС» – ссылка на элемент справочника, значение автоматически выбирается из связанного справочника «ЕКПС» по первым двум знакам класса ЕКПС;

- «Ном. номер ВМФ» – номенклатурный номер ВМФ («Юпитер-СТО»), символьное значение длиной 10 символов вводится с клавиатуры;

- «ОКПД2» – ссылка на элемент справочника, значение выбирается из связанного справочника «ОКПД2»;

- «ТН ВЭД» – ссылка на элемент справочника, значение выбирается из связанного справочника «ТН ВЭД»;

- «Версия прошивки» – версия прошивки памяти / версии ПО, символьное значение длиной не более 30 символов;

- «Дата прошивки» – дата версии прошивки памяти / версии ПО, короткий формат даты. Значение вводится с клавиатуры или выбирается из встроенного в поле ввода календаря;

- «Статус» – ссылка на элемент справочника, значение выбирается из связанного справочника «Статусы записей справочников и каталогов», обязательно поле;

«Год снятия с производства» – цифровое значение длиной 4 знака вводится с клавиатуры

На вкладке «Характеристики» формируется перечень характеристик, описывающих создаваемый ПС. Для добавления новой характеристики ПС необходимо выполнить следующую последовательность действий:

- нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Характеристики», в результате создастся строка с порядковым номером для ввода характеристики. Номер характеристики создастся автоматически на основе счетчика. Изменение порядка следования характеристик в списке выполняется с помощью кнопок перемещения.

- в поле «Характеристика» открыть связанный справочник «Характеристики предметов снабжения»;

- выбрать необходимый элемент в открывшейся форме связанного справочника;

- выбрать тип значения характеристики из связанной формы «Выбор типа данных» – число или строка;

- заполнить значение характеристики в соответствии с выбранным типом данных. Цифровое значение – длиной не более 13 символов (10 знаков на целую часть и 2 знака на дробную);

- выбрать значение единицы измерения характеристики из связанного справочника «ОКЕИ»;

- указать значение признака «Основная» для характеристики – булевое значение, необязательный признак;

- указать значение признака «Отображение для заказчика» – булевое значение, необязательный признак.

 

На вкладке «Правила поставки» в текстовом поле вводом с клавиатуры указываются правила упаковки, транспортировки и хранения редактируемого ПС

На вкладке «Поставщики» формируется перечень поставщиков редактируемого ПС

Для добавления нового поставщика необходимо выполнить следующую последовательность действий:

- нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Поставщики», в результате создастся строка с порядковым номером для ввода контрагента. Номер поставщика создастся автоматически на основе счетчика. Изменение порядка следования элементов в списке выполняется с помощью кнопок перемещения;

- в поле «Контрагент» открыть связанный справочник «Организации»;

- выбрать необходимый элемент в открывшейся форме связанного справочника, в результате заполнится поле «Контрагент» и автоматически заполнится поле «Город»;

- установить в поле «Поставщик» признак поставщика ПС, в случае если выбранный контрагент соответствует данному признаку;

- установить в поле «Разработчик» признак разработчика ПС, в случае если выбранный контрагент соответствует данному признаку;

- установить в поле «Изготовитель» признак изготовителя ПС, в случае если выбранная организация соответствует данному признаку;

- в поле «Сроки изготовления» указать срок изготовления редактируемого ПС выбранным контрагентом;

- в поле «Единица измерения» выбрать из связанного справочника «ОКЕИ» значение, соответствующее редактируемому ПС;

в поле «Состояние производства» выбрать из связанного справочника «Состояния производства ПС» значение, соответствующее редактируемому ПС. 

На вкладке «Аналоги» формируется перечень ПС, являющихся аналогами редактируемого ПС. Для добавления нового аналога ПС необходимо выполнить следующую последовательность действий:

- нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Аналоги», в результате создастся строка с порядковым номером для ввода наименования ПС. Номер ПС создастся автоматически на основе счетчика. Изменение порядка следования элементов в списке выполняется с помощью кнопок перемещения;

- в поле «Аналог» открыть связанный каталог ПС;

- выбрать необходимый элемент в открывшейся форме связанного справочника;

выбрать тип замены в одноименном поле из связанного справочника «Типы замены ПС аналогом».

 

 

На вкладке «Цены внешние» отображается список внешних цен редактируемого ПС, сформированный автоматически на основании данных различных объектов ИСС ЗИП-Э.

На вкладке «Цены внутренние» отображается список внутренних цен редактируемого ПС, сформированный автоматически на основании данных различных объектов ИСС ЗИП-Э.

 

На вкладке «Состав» формируется структура редактируемого ПС.

Для добавления нового составляющего элемента ПС необходимо нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Состав», и в открывшейся форме создания составной части ПС «Форма позиции состава» заполнить следующие поля:

- «Составная часть» – ссылка на элемент справочника, значение выбирается из связанного каталога ПС, обязательное поле;

- «Обозначение» – заполняется автоматически на основании выбора составной части ПС, в случае заполнения данного поля в каталоге ПС;

- «ЗИП» – установить признак ЗИП, в случае если выбранная составная часть ПС соответствует данному признаку;

- «Тип» – ссылка на элемент справочника, значение выбирается из связанного справочника «Разделы спецификации ЗИП», обязательное поле;

- «Количество» – количество составных частей в ПС вышестоящего уровня, цифровое значение длиной не более 3 знаков вводится с клавиатуры, обязательное поле;

- «Обозначение СНК» – иерархический уровень составной части в соответствии с используемой системой нумерации;

- «Причина выбора для ЗИП» – ссылка на элемент справочника, значение выбирается из связанного справочника «Причины выбора ПС для ЗИП», обязательное поле;

- «Указатель на элемент схемы» – условное обозначение составной части на схеме, символьное значение длиной не более 50 символов вводится с клавиатуры;

- «Взаимозаменяемость» – указание взаимозаменяемости и предпочтительности ПС, устанавливаемых на одно место в ПС вышестоящего уровня. Символьное значение длиной не более 150 символов вводится с клавиатуры;

- «Признак подбора» – ссылка на элемент справочника, значение выбирается из связанного справочника «Признаки подбора или изготовления ПС»;

- «Обозначение заготовки» – обозначение заготовки  или ремонтного комплекта для изготовления составной части, символьное значение длиной не более 200 символов вводится с клавиатуры;

«Сведения об изменениях» – номера изменений или извещений об изменениях, символьное значение длиной не более 50 символов.

ПС, добавляемые в структуру в качестве составных частей, в свою очередь также могут состоять из составных частей. Просмотр всей структуры состава редактируемого ПС выполняется нажатием кнопки « Показать всю структуру». В результате нажатия кнопки «Показать всю структуру» открывается окно «Вся структура предмета снабжения», содержащее перечень ПС, входящих в качестве составных частей в ПС, перечисленные на вкладке «Состав».

 

На вкладке «Ресурсы и сроки службы» формируется список показателей учета ресурса и срока службы редактируемого ПС.

Для добавления новой записи показателя ресурса и срока службы необходимо выполнить следующую последовательность действий:

  нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Ресурсы и сроки службы», в результате чего создастся строка с порядковым номером для ввода показателя. Номер параметра учета ресурса и срока службы создастся автоматически на основе счетчика. Изменение порядка следования элементов в списке выполняется с помощью кнопок перемещения;

- в поле «Показатель» открыть связанный справочник «Типы показателей изделий»;

- выбрать необходимый элемент в открывшейся форме связанного справочника;

- в поле «Параметр учета» открыть связанный справочник «Параметры учета показателей»;

- выбрать необходимый элемент в открывшейся форме связанного справочника;

- в поле «Значение» задать значение показателя применительно к выбранному параметру учета;

в поле «Единица измерения» задать значение единицы измерения показателя применительно к выбранному параметру учета.

 

На вкладке «Регламент ТО» формируется перечень работ технического обслуживания (ТО) и ремонта (Р) редактируемого ПС. Данные по регламенту ТО разделены на две группы:

- «Регламент ТО и Р», в которой определяется список требуемых работ;

- «Список ЗИП, требуемых для ТО», в которой определяется список ЗИП, необходимых для проведения работ, указанных в регламенте ТО.

Для добавления новой записи в перечень работ ТО и Р необходимо выполнить следующую последовательность действий:

- нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Регламент ТО и Р», в результате создастся строка с порядковым номером для ввода вида работ. Номер элемента в перечне работ ТО и Р создастся автоматически на основе счетчика. Изменение порядка следования элементов в списке выполняется с помощью кнопок перемещения;

- в поле «Вид работ» открыть связанный справочник «Виды ремонта»;

- выбрать необходимый элемент в открывшейся форме связанного справочника;

- в поле «Периодичность» задать значение периодичности проведения указанной работы;

- в поле «Ед. изм. периодичности» выбрать из связанного справочника «ОКЕИ» значение, соответствующее заданной периодичности проведения работы;

- в поле «Продолжительность» задать продолжительность проведения выбранной работы в виде цифрового значения длиной не более 11 знаков (8 знаков на целую часть и 2 знака на дробную часть);

- в поле «Ед. изм. продолжительности» выбрать из связанного справочника «ОКЕИ» значение, соответствующее заданной продолжительности проведения работы;

- в поле «Трудоемкость работы» задать трудоемкость выбранной работы в нормо-часах в виде цифрового значения длиной не более 11 знаков (8 знаков на целую часть и 2 знака на дробную часть);

- в поле «Трудоемкость сопутствующих работ» задать трудоемкость сопутствующих работ, по отношению к выбранной работе,  в нормо-часах в виде цифрового значения длиной не более 11 знаков (8 знаков на целую часть и 2 знака на дробную часть).

 

Для добавления новой записи в список требуемых для ТО ЗИП необходимо выполнить следующую последовательность действий:

- нажать кнопку «Добавить» или выбрать одноименный пункт контекстного меню в группе «Список ЗИП, требуемых для ТО», в результате создастся строка с порядковым номером для ввода наименования ЗИП. Номер добавляемого ЗИП создастся автоматически на основе счетчика. Изменение порядка следования элементов в списке выполняется с помощью кнопок перемещения;

- в поле «ЗИП» открыть связанный каталог ПС;

- выбрать необходимый элемент в открывшейся форме связанного справочника, в результате заполняется поле «ЗИП» и автоматически заполняется поле «Обозначение», в случае заполнения данного поля в каталоге ПС;

- в поле «Количество» задать количество выбранного элемента ЗИП в виде цифрового значения длиной не более 11 знаков (8 знаков на целую часть и 2 знака на дробную часть);

в поле «Ед. изм.», в случае необходимости, откорректировать значение, введенное автоматически из каталога ПС в соответствии с выбранным ЗИП, или выбрать из связанного справочника «ОКЕИ» значение единицы измерения количества ЗИП, соответствующее выбранному ЗИП.

 

На вкладке «Служебные данные идентификации и кодирования». заполняются следующие поля информацией, полученной в результате выполнения бизнес-процесса «Взаимодействие с РОЭ»:

- «CIN» – уникальный порядковый номер ПС в БД каталогизированной продукции военного назначения. Цифровое значение длиной не более 18 знаков загружается из файла АО «РОЭ» или вводится вручную;

- «Дата создания» – дата создания записи в каталоге АО «РОЭ». Короткий формат даты. Значение загружается из файла АО «РОЭ» или вводится с клавиатуры, или выбирается из встроенного в поле ввода календаря;

- «Дата присвоения» – дата присвоения номера ПС в каталоге АО «РОЭ». Короткий формат даты. Значение загружается из файла АО «РОЭ» или вводится с клавиатуры, или выбирается из встроенного в поле ввода календаря;

- «Дата отмены» – дата отмены номера ПС в каталоге  АО «РОЭ». Короткий формат даты. Значение загружается из файла АО «РОЭ» или вводится с клавиатуры, или выбирается из встроенного в поле ввода календаря;

- «TIIC» – метод идентификации ПС. Символьное значение длиной не более 2 символов загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RPDMRC» – причина выбора ссылочного или частично-описательного метода идентификации ПС. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «Кодификатор» – исполнитель кодификации ПС в каталоге АО «РОЭ». Символьное значение длиной 2 символа загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNCC» – код категории ссылочного номера. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNVC» – код варианта ссылочного номера. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «DAC» – код доступности документации для АО «РОЭ». Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNJC» – код обоснования идентификации ПС по ссылочному номеру. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNSC» – код статуса ссылочного номера. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNFC» –  код формата ссылочного номера. Символьное значение длиной 1 символ загружается из файла АО «РОЭ» или вводится с клавиатуры;

- «RNAAC» – код организации, которая управляет ссылочным номером. Символьное значение длиной 4 символа загружается из файла АО «РОЭ» или вводится с клавиатуры.
